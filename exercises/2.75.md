## Exercise 2.75

Implement the constructor `make-from-mag-ang` in message-passing style. This
procedure should be analogous to the `make-from-real-imag` procedure given
above.

## Solution

```scheme
(define (make-from-mag-ang r a)
  (define (dispatch op)
    (cond ((eq? op 'real-part) (* r (cos a)))
          ((eq? op 'imag-part) (* r (sin a)))
          ((eq? op 'magnitude) r)
          ((eq? op 'angle) a)
          (else
            (error "Unknown op:
                   MAKE-FROM-MAG-ANG" op))))
   dispatch)
```

We could even optimize the code by avoiding to recompute `(cos a)` and `(sin a)`
on every call:

```scheme
(define (make-from-mag-ang r a)
  (define real-part (* r (cos a)))
  (define imag-part (* r (sin a)))
  (define (dispatch op)
    (cond ((eq? op 'real-part) real-part)
          ((eq? op 'imag-part) imag-part)
          ((eq? op 'magnitude) r)
          ((eq? op 'angle) a)
          (else
            (error "Unknown op:
                   MAKE-FROM-MAG-ANG" op))))
   dispatch)
```

Now instead of computing real/imag on demand, we precompute and store real/imag
in the closure.

Note that this is a space-time tradeoff, not a free win, because we need to
store two extra values.
